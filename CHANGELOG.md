# 更新日志 (Changelog)

按日期记录项目重要变更。

## 2026-07-09 — v1.1.1 稳定性与兼容性更新

### Quilt 加载器原生支持

- **新增 QuiltLoader 类**：完全兼容 Quilt 模组加载器，复用 Fabric 库解析逻辑，无需额外适配
- **版本自动识别**：`cli.py` 中根据文件夹名自动区分 Fabric 与 Quilt 版本
- **实测通过**：Quilt 0.29.2 + Minecraft 1.20.1 正常启动，模组列表完整加载

### 崩溃检测与状态监控修复

- **修复状态检查异常**：`crash_detector.is_running()` 增加 `subprocess.Popen` 类型检查，避免 `bool` 对象调用 `poll()` 导致闪退
- **GUI 异常捕获**：`_check_game_status` 方法增加 `try-except` 包裹，防止未捕获异常导致启动器崩溃
- **进程正确传递**：`launch_selected_version` 返回进程对象，`LaunchWorker` 正确传递给 `finished_signal`

### 资源下载与启动优化

- **优化资源下载逻辑**：多线程异步下载，实时进度反馈
- **Asset 索引自动解析**：首次启动自动下载缺失资源文件
- **NeoForge 兼容性**：完善 NeoForge 47.1.106 启动参数传递，确保模组加载器正常工作

### 其他改进

- **路径导入修复**：修正 `core/cli.py` 中 `PROJECT_ROOT` 路径层级，确保模块正确导入
- **编码兼容**：GUI 入口强制设置 `PYTHONIOENCODING=utf-8`，避免中文路径编码问题

## 2026-07-08 — v1.1.0 重大功能更新

### GUI 图形界面（队友贡献，基于 PyQt5）

- **新增 PyQt5 图形界面**：完全替代命令行操作，提供可视化交互体验
- **版本管理**：支持浏览、选择、下载 Minecraft 游戏版本（原版/Forge/Fabric/NeoForge）
- **服务器管理**：一键下载服务端核心、启动/停止本地服务器、实时控制台日志、发送命令
- **账号管理**：支持离线账号创建与切换，预留微软正版登录接口
- **模组管理**：模组列表查看与启用/禁用功能
- **下载引擎**：多线程异步下载，实时显示进度条
- **主题适配**：统一 UI 风格，卡片式布局，清晰易用

### EasyTier P2P 联机功能

- **新增 EasyTier P2P 联机功能**：基于点对点技术，无需服务器中转，完全免费，无需实名认证，不限年龄
- **支持房主/加入双模式**：房主输入房间名和密码开房，朋友输入相同信息加入
- **智能 IP 获取**：自动识别虚拟 IP，支持备选 IP 轮询
- **防火墙引导**：启动联机前自动检测防火墙状态并提示用户
- **一键复制地址**：复制 `IP:端口` 到剪贴板，方便分享给朋友
- **自定义端口**：支持用户自定义游戏端口，默认 25565
- **完整用户引导**：加入房间后弹出图文教程，指导玩家在游戏内连接

## 2026-07-07 — v1.0.3 内核版本更新

- **新增模组加载器支持（模块化封装）**：抽象出 `BaseLoader` 基类，独立封装 `vanillaLoader`、`ForgeLoader`、`NeoForgeLoader`、`FabricLoader`，代码结构更清晰、可扩展性更强
- **新增三款主流模组加载器支持**：`Forge`、`Fabric`、`NeoForge` 均可正常启动
- **兼容 OptiFine**：支持 OptiFine 独立版（安装后生成独立版本 JSON）和 Forge 版（作为普通模组加载），无需额外适配
- **修复 Fabric 库解析异常**：兼容 Fabric 版本 JSON 中的 `name` 格式库，彻底解决 `KeyError: 'downloads'` 问题
- **解决 Native 库多版本冲突**：移除 `--module-path`，采用纯类路径启动，彻底消除模块冲突
- **优化参数透传**：统一替换 JvM 参数和游戏参数中的所有 `${...}` 占位符，确保所有加载器参数正确传递
- **AI项目重构**：借助AI重构、优化项目结构，所有重构内容均由团队人工检查、核对并且修改(此操作并非全部重构，接口任何变动)
- **重命名启动入口**：将 `launch.py` 重命名为 `cli.py`
