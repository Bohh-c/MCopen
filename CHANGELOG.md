## 2026-07-11 — v1.4.3 GUI 全面重构与联机方案迁移
### UI 全面重构
- **整体布局从卡片式改为列表式**：所有页面统一为简约风格
- **窗口尺寸固定**：主窗口锁定 1080x720，不允许用户随意拖动改变大小
- **设置页面独立**：将 Java 路径、内存分配、游戏语言从主窗口移至独立设置页，首页仅保留版本选择和启动，优化布局
- **移除所有按钮宽度/高度限制**：按钮完全由 Qt 布局自动分配空间，文字不再被截断
- **所有页面统一重构**：首页、下载、加载器、模组管理、模组下载、辅助联机、工具箱、服务器、账号、日志、设置全部按新风格重构
### 联机方案迁移
- **移除 EasyTier P2P 联机**：因网络环境兼容性差、打洞成功率不稳定，完全移除 EasyTier
- **新增 ZeroTier 虚拟局域网联机**：采用 ZeroTier 作为主力联机方案，用户需自行安装 ZeroTier 客户端
- **ZeroTier 联机页面重构**：简化操作流程，输入 Network ID 即可加入网络，自动轮询获取虚拟 IP
### 加载器功能增强
- **Forge/NeoForge 换源**：从 BMCLAPI 迁移至官方 Maven 源，版本列表实时同步官方最新版本（47.4.x）
- **安装进度显示**：新增状态标签显示当前安装操作详情，不再卡在 0%
- **安装失败智能提示**：版本不可用时自动推荐可用版本列表
### 启动器体验优化
- **Java 路径持久化**：设置保存后永久生效，不再每次启动重新选择
- **子进程黑框彻底隐藏**：使用 `CREATE_NO_WINDOW` 标志，打包后无多余控制台窗口
- **启动页面简化**：仅保留版本选择、玩家名切换、启动按钮，布局清爽

## 2026-07-10 — v1.4.2 联机稳定性与打包优化
### 修复离线账号无效会话问题
- **离线令牌格式修正**：将 `accessToken` 改为 `"0"`，`userType` 改为 `"legacy"`，使 Minecraft 1.20.1 客户端正确识别为离线用户，彻底解决“无效会话”登录失败问题
- **UUID 动态生成**：修复 `AccountManager.create_offline` 中 UUID 固定导致“此名称已被占用”的问题，每次启动生成随机 UUID，多人联机不再冲突
### e4mc 联机连接优化
- **域名直连支持**：修复 e4mc 托管后朋友无法连接的问题，主机端日志显示 `Domain assigned` 即表示隧道已打通
- **连接错误排查指引**：针对“无效会话”、“此名称已被占用”、“无法验证聊天消息”等常见报错，提供清晰的排查方向
### 启动画面体验优化
- **新增启动画面**：使用 `1.png` 作为启动界面，主窗口加载完成前显示，提升启动体验
- **启动速度优化**：使用 `QTimer.singleShot(0, ...)` 将耗时初始化延迟到事件循环启动后执行，启动画面不再卡顿
- **纯色备用方案**：当 `1.png` 不存在或加载失败时，自动降级为纯色启动界面，避免空白
### 打包方案更新
- **从 PyInstaller 迁移至 Nuitka**：生成更小、启动更快的独立 exe 文件
- **编译器适配**：针对 Python 3.14，打包方案从 MinGW 切换至 MSVC（Visual Studio Build Tools）
- **无控制台窗口**：使用 `--windows-console-mode=disable` 彻底隐藏命令行黑框
- **资源文件打包**：正确包含 `core/`、`gui/`、`tools/` 目录及 `1.png`、`1.ico` 文件
- **【新增补充】**当前打包输出exe文件名仍硬编码为v1.4.0，版本号同步问题待下一补丁版本统一修复
### 版本兼容性说明
- **1.20.1 及以下版本**：当前启动器方案完全兼容，支持离线联机
- **1.20.2 及以上版本**：需要集成 `authlib-injector` 以支持新版公钥验证机制，将在后续版本中实现
### 项目与宣发补充
- **【新增补充】**完成两支B站项目预热宣传片，当前Gitee仓库暂未公开，仅对内测小范围分发测试包
- **【新增补充】**确定项目永久开源协议为 GNU GPL v3.0，后续开源时同步附上完整LICENSE、AI_NOTICE.md合规文档
